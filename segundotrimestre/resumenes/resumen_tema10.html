Resumen de XML

XML (Extensible Markup Language) es un lenguaje de marcado desarrollado por el W3C con el propósito de compartir datos a través de diferentes sistemas, como Internet.

Características de XML

  • No está predefinido, las etiquetas se definen según las necesidades.
  • Constituye un lenguaje de estructura libre, formado por un prólogo y un cuerpo.
  • Debe cumplir con las reglas del W3C para estar bien formado.
  • Utiliza esquemas en forma de árbol con una única raíz.
  • Se distingue de HTML en que no tiene etiquetas predefinidas y debe cumplir normas estrictas de cierre y anidación.
  • Usa espacios de nombres para evitar ambigüedades.
  • Permite el uso de hojas de estilo CSS y XSLT para presentación.

Estructura de un Documento XML

  • Prólogo: Contiene la declaración XML y puede incluir información sobre el documento.
  • Cuerpo: Organizado en estructura de árbol con una única raíz.

Sintaxis XML

  • Las etiquetas deben estar bien formadas y cerradas correctamente.
  • Los nombres de las etiquetas distinguen entre mayúsculas y minúsculas.
  • Los atributos deben estar entre comillas simples o dobles.
  • Los espacios en blanco no afectan la estructura del documento.

Entidades en XML

XML permite el uso de entidades para representar caracteres especiales:

  • &lt; representa < (menor que).
  • &gt; representa > (mayor que).
  • &amp; representa & (ampersand).
  • &quot; representa " (comilla doble).
  • &apos; representa ' (comilla simple).

Validación XML

Para validar un documento XML, se pueden usar:

  • DTD (Document Type Definition): Un metalenguaje que permite la descripción del formato que deben cumplir los datos en un documento XML.
  • XML Schema: Lenguaje basado en XML que mejora el DTD, permitiendo definir con más detalle los tipos de datos.

Tipos de Validación con DTD

  • DTD Interno: Definido dentro del propio documento XML.
  • DTD Externo: Referenciado mediante una URL.
  • DTD Mixto: Combinación de ambos métodos.

Tipos de Elementos DTD

  • Elemento: <!ELEMENT>
  • Atributos: <!ATTLIST>
  • Entidades: <!ENTITY>

Cardinalidad

  • Cardinalidad: Número de veces que aparece el elemento descendiente
    ?: El elemento aparece 0 o 1 vez.
    *: El elemento aparece 0 a N veces.
    +: El elemento aparece 1 a N veces.
  • Secuencia: Se indican los elementos que deben de aparecer en el fichero XML.
    X, Y: Indica que el elemento X tiene que aparecer a continuación de Y.
    (X|Y): Indica que aparece el elemento X o el elemento Y, pero no los dos al mismo tiempo.
  • Combinación: Se pueden combinar los símbolos de secuencia y los de cardinalidad.

Ejemplo de Código XML

<?xml version="1.0" encoding="UTF-8"?>
<biblioteca>
    <libro>
        <titulo>Ejemplo de XML</titulo>
        <autor>Autor Ejemplo</autor>
        <capitulo>
            <titulocap>Capítulo 1</titulocap>
            <texto>Contenido del capítulo 1</texto>
        </capitulo>
    </libro>
</biblioteca>
